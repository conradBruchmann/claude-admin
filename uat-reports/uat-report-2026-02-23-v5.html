Refactoring-Verifikation (7 Phasen)

Jede Phase wurde einzeln verifiziert: cargo fmt, cargo clippy, cargo test nach jedem Schritt.

Phase 1: Shared Route Builder

Vorher: 255 Zeilen Route-Chain in app.rs + 155 Zeilen Duplikat in Tests Nachher: 311 Zeilen in router.rs, 1 Zeile in app.rs + Tests Ergebnis: Route-Drift zwischen Prod und Test eliminiert

Phase 2: Middleware-Modernisierung

from_fn_with_state statt Extension-Workaround

Vorher: Extension(token_store), Extension(rate_limiter), Extension(claude_home) Nachher: State(Arc<AppState>) direkt in auth_middleware + rate_limit_middleware Ergebnis: 3 Extension-Layers eliminiert, typsicherer Zugriff

Phase 3: RBAC Caching

Arc<tokio::sync::RwLock<RbacConfig>> in AppState

Vorher: RbacConfig::load() liest users.json synchron bei JEDEM Request Nachher: Cache im Speicher, Auto-Reload via File-Watcher bei Aenderung Ergebnis: Keine sync I/O im async Context mehr

Phase 4: Sichere Tokens

HMAC-SHA256 statt SystemTime-LCG

Vorher: ses_{timestamp_hex}_{lcg_hex} (vorhersagbar) Nachher: ses_{32 hex chars} (HMAC-SHA256 mit Process-Secret + Counter) Ergebnis: 128-Bit Entropie, kryptographisch sicher

Phase 5: Dead Code Cleanup

Zero Compiler-Warnings
